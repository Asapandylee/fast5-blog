처음 MCP 얘기를 들었을 때 솔직히 귀에 안 들어왔습니다. "AI가 외부 도구랑 연결되는 프로토콜이야" — 이게 무슨 말인지 알겠는데 실제로 뭐가 달라지는지 감이 안 왔거든요.

직접 써보고 나서야 "아, 이게 이런 거구나"가 됐습니다. 같은 과정을 겪고 있는 분들한테 도움이 됐으면 해서 제가 이해한 방식으로 설명해봅니다.

## MCP를 한 문장으로

**AI가 당신의 컴퓨터, 앱, 데이터에 직접 접근할 수 있게 해주는 표준 연결 방식**입니다.

좀 더 풀면: 원래 ChatGPT나 Claude한테 "내 구글 캘린더 일정 알려줘"라고 하면 당연히 못 합니다. AI가 내 캘린더에 접근하는 방법이 없으니까요. MCP는 이 연결을 표준화된 방식으로 만들어주는 겁니다.

## 플러그인이랑 다른 게 뭔데?

ChatGPT도 플러그인 있었잖아요. 그거랑 뭐가 다르냐고 물으면 — 결국 비슷한 개념인데 MCP는 **개방형 표준**이라는 게 핵심입니다.

특정 회사가 만들어서 자기들 AI에만 쓰는 게 아니라, Anthropic이 표준을 공개했고 누구나 MCP 서버를 만들 수 있어요. Claude, Cursor, Windsurf 등 여러 AI 툴이 같은 MCP 서버를 공유해서 쓸 수 있습니다.

이게 왜 중요하냐면 — 한번 만들어두면 여러 AI 툴에서 다 쓸 수 있다는 거예요.

## 실제로 쓰면 뭐가 달라지나

제가 Claude Code에 Notion MCP를 연결해서 써봤을 때의 경험입니다.

연결 전: "내 Notion에 있는 프로젝트 목록 보면서 TODO 정리해줘" → 불가능

연결 후: Claude Code가 Notion에 직접 접근해서 페이지 읽고, 새 페이지 만들고, 내용 업데이트까지 합니다. 제가 따로 복사붙여넣기 안 해도 됩니다.

이거 한번 경험하면 MCP가 왜 중요한지 바로 이해됩니다.

## 지금 쓸 수 있는 MCP 서버들

2025년 초 기준으로 꽤 많아졌습니다:

- **파일시스템**: 로컬 파일 읽기/쓰기
- **GitHub**: 저장소 접근, 이슈 관리
- **Notion**: 페이지 읽기/쓰기
- **Slack**: 메시지 전송, 채널 읽기
- **브라우저**: 웹페이지 스크래핑
- **데이터베이스**: PostgreSQL, SQLite 직접 쿼리

Claude Code를 쓰면 `claude mcp add`로 간단하게 추가할 수 있어요. 설정 자체는 어렵지 않습니다.

## MCP가 바꿀 것들

개인적으로 MCP가 가장 크게 바꿀 거라고 생각하는 건 **워크플로우 자동화**입니다.

지금은 AI랑 대화하고 → 그 내용을 내가 직접 다른 앱에 입력하는 과정이 있잖아요. MCP가 보편화되면 이 중간 과정이 사라집니다. AI가 알아서 캘린더 업데이트하고, Notion 정리하고, GitHub 이슈 만들고.

아직 초기 단계라 어색한 부분도 있지만, 방향은 확실한 것 같아요. 1~2년 안에 AI와 앱의 경계가 많이 흐려질 거라고 봅니다.
