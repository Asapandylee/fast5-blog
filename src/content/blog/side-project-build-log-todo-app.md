Todo 앱은 "Hello World" 다음으로 많이 만드는 앱이라고 하죠. 저도 예전에 공부할 때 만들어봤는데, 그때랑 지금이 완전히 다른 경험이었습니다.

이번엔 진짜 쓸 수 있는 앱을 목표로 잡았어요. 그냥 체크박스 앱이 아니라, 반복 태스크, 우선순위, 태그 필터링, 간단한 통계까지. 주말 이틀 동안 만들어봤습니다.

## 시작 전 세팅: 스택 정하기

스택은 빠르게 정했습니다.

- **프론트엔드**: Next.js 14 (App Router)
- **스타일**: Tailwind CSS
- **백엔드**: Supabase (DB + 인증)
- **배포**: Vercel

이 스택을 선택한 이유는 AI가 가장 잘 아는 조합이기 때문이에요. 마이너한 프레임워크 쓸수록 AI 답변 퀄리티가 떨어지는 걸 경험해봤습니다. 익숙한 스택 = AI가 더 잘 짜준다는 게 제 경험칙입니다.

## 토요일 오전: 기획 30분

To-do 앱이지만 기획은 필요합니다. 아무리 간단해도 "이거 만들어줘"만 하면 나중에 방향이 흔들려요.

Claude한테 "일반 Todo 앱이랑 다르게 만들고 싶은데, 차별점 아이디어 뭐가 있을까?"라고 물어봤습니다. 여러 아이디어 중에 **"완료한 태스크를 분석해서 패턴 보여주는 기능"** 이 마음에 들어서 이걸 메인 차별점으로 잡았습니다.

![프로젝트 계획과 분석](https://images.unsplash.com/photo-1551288049-bebda4e38f71?w=1200&q=80)

## 토요일 오후: 핵심 기능 구현

Supabase 세팅하고 스키마 짜는 데 1시간 썼습니다. Claude한테 "이런 기능이 필요한데 테이블 구조 어떻게 잡으면 좋을까?"라고 물어보면서 같이 설계했어요.

처음 제안한 구조에서 제가 "반복 태스크를 어떻게 저장하지?"라는 문제를 발견했는데, Claude가 `recurrence_pattern` JSON 컬럼으로 처리하는 방법을 제안해줬고, 그게 꽤 깔끔하게 동작했습니다.

오후 내내 컴포넌트 만들었는데, 여기서 Cursor가 빛났어요. 하나의 컴포넌트 완성하면 비슷한 컴포넌트는 Tab 자동완성으로 뚝딱 나왔습니다.

## 일요일: 버그 잡기 + 배포

일요일 오전은 예상대로 버그 잡는 날이었어요. 가장 오래 걸린 버그가 뭔지 아세요? **날짜 관련 버그**였습니다. 타임존 처리가 엉켜서 한국 시간 기준으로 오늘 추가한 태스크가 어제 날짜로 저장되는 문제가 있었어요.

이거 Claude한테 물어보니까 바로 원인 찾아줬습니다. Supabase가 UTC로 저장하는데 클라이언트에서 로컬 시간으로 비교하고 있었던 거였어요. 이 버그 혼자 찾았으면 몇 시간은 걸렸을 것 같습니다.

오후에 Vercel 배포하고 도메인 연결. 총 소요 시간 약 14시간.

## 느낀 점

이틀 만에 이 정도 퀄리티가 나온 게 AI 없었으면 불가능했을 거라고 확신합니다. 예전이라면 2~3주는 잡았을 프로젝트거든요.

다만 내 코드가 아닌 느낌도 살짝 있어요. AI가 많이 짜줬으니까요. 이건 아직도 익숙해지는 중입니다. 내가 설계하고 AI가 구현한다는 마인드로 접근하면 좀 더 오너십이 생기는 것 같아요.
