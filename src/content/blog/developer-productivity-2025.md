매년 "개발자 생산성 도구" 글이 쏟아지는데, 솔직히 대부분 비슷비슷합니다. Notion, Figma, Slack... 다 아는 것들이죠.

이 글은 그런 얘기가 아닙니다. 제가 실제로 매일 쓰면서 체감 속도가 올랐다고 느끼는 것들만 추렸어요. 화려하거나 트렌디하지 않아도 됩니다. 그냥 빠르면 됩니다.

## 1. Raycast (맥 전용)

맥의 Spotlight를 대체하는 런처인데, 이게 없던 시절로 돌아가기가 너무 힘들어졌습니다.

앱 실행, 클립보드 히스토리, 파일 검색, 계산기, 번역, 심지어 ChatGPT 연동까지 — 키보드 하나로 다 됩니다. `cmd+space` 하나로 컨텍스트 스위칭 없이 뭐든 할 수 있다는 게 생각보다 시간을 많이 아껴줘요.

특히 **Clipboard History**는 진짜 필수입니다. 예전 복사한 내용을 다시 불러올 수 있는데, 코딩할 때 여러 개 복붙을 동시에 해야 할 때 이게 없으면 답답해서 못 삽니다.

## 2. Warp (터미널)

"터미널이 뭐가 달라봤자 얼마나 다르겠어"라고 생각했는데 Warp 쓰고 생각이 바뀌었습니다.

명령어 자동완성, AI 질의, 실행한 명령어 블록 단위로 관리 — 특히 AI한테 "이거 어떻게 하는 명령어야?"라고 물어보면 바로 명령어로 대답해줍니다. 터미널에서 구글 안 열어도 되는 게 생각보다 편해요.

## 3. GitHub Copilot 채팅 (VSCode 내장)

Cursor까지 돈 내기 부담스러운 분들한테 추천합니다. VSCode 쓰면서 Copilot 채팅 기능을 잘 활용하면 꽤 쓸 만합니다.

`@workspace`로 프로젝트 전체 컨텍스트를 참조할 수 있고, `@terminal`로 터미널 에러를 바로 분석해달라고 할 수 있어요.

## 4. Obsidian (개발 노트)

Notion을 쓰다가 갈아탔는데, 가장 큰 이유는 **로컬 파일 기반**이라는 것. 마크다운 파일이 내 컴퓨터에 있으니까 영원히 내 거라는 안심감이 있습니다.

개발하면서 에러 해결한 거, 새로 배운 개념, 나중에 찾아볼 링크 — 이런 걸 날짜별로 쌓아두면 나중에 엄청 유용해요. 저는 6개월치 개발 노트가 지금 제일 가치 있는 자산인 것 같습니다.

## 5. Linear (이슈 트래킹)

팀이 있다면 Jira 대신 Linear를 추천합니다. 속도가 다릅니다. 같은 작업을 Jira에서 하면 클릭 5번, Linear에서 하면 단축키 2번이에요.

혼자 사이드프로젝트 할 때도 Linear 씁니다. 할 일 관리가 깔끔하게 되고, GitHub 연동하면 코드 커밋이랑 이슈가 자동으로 연결됩니다.

## 6. Excalidraw

다이어그램 그리는 툴인데, 왜 이게 생산성 도구냐 하면 — 코딩 전에 구조를 손으로 그려보는 시간이 결국 코딩 시간을 줄여주거든요. AI한테 설명할 때도 다이어그램 만들어서 붙여넣으면 훨씬 정확하게 이해합니다.

## 7. .cursorrules 또는 CLAUDE.md

AI 툴을 쓸 때 프로젝트마다 컨텍스트 파일 만드는 습관. 기술 스택, 코딩 컨벤션, 하지 말아야 할 것들을 미리 적어두면 매번 설명 안 해도 됩니다. 이게 개발 속도에 생각보다 크게 기여해요.

---

생산성 도구는 결국 "얼마나 자주 쓰냐"가 핵심입니다. 아무리 좋은 툴도 안 쓰면 0이에요. 위에 적은 것들은 제가 진짜 매일 쓰는 것들이니, 하나씩 써보고 자기한테 맞는 걸 남기는 방식으로 접근하면 좋을 것 같습니다.
