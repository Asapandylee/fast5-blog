Cursor를 먼저 쓰기 시작했고, Claude Code는 나온 지 얼마 안 됐을 때 갈아탔습니다. 지금은 둘 다 씁니다. 용도가 다르거든요.

이걸 진작에 알았으면 초반에 "어떤 게 더 낫나" 고민하는 시간을 아꼈을 텐데 싶어서, 직접 겪은 걸 기반으로 정리해봤습니다.

## Cursor: IDE 자체가 AI가 된 경험

Cursor는 VSCode 기반 에디터입니다. 겉보기엔 그냥 VSCode인데, AI가 코드베이스 전체를 인덱싱해서 맥락을 유지하면서 코드를 짜줍니다.

처음 썼을 때 가장 놀랐던 건 **Tab 자동완성**이었어요. 그냥 다음 줄 예측이 아니라, 내가 지금 만들고 있는 함수의 흐름을 이해하고 통째로 완성해주는 느낌? 몇 글자 치면 나머지를 채워주는데, 이게 맞을 때의 쾌감이 꽤 중독성 있습니다.

**Composer 모드**는 진짜 바이브코딩의 핵심입니다. "이 컴포넌트에 다크모드 추가해줘"라고 하면 관련 파일 여러 개를 동시에 수정해줍니다. diff로 보여주니까 뭐가 바뀌었는지 확인하기도 쉽고요.

![개발 환경 비교](https://images.unsplash.com/photo-1551288049-bebda4e38f71?w=1200&q=80)

## Claude Code: 터미널에서 AI가 프로젝트를 직접 만진다

Claude Code는 터미널 기반입니다. IDE가 아니에요. `claude` 명령어로 실행하면 AI랑 대화하듯이 작업을 시키는데, 파일을 직접 읽고, 수정하고, 생성하고, 심지어 git commit까지 합니다.

처음엔 "터미널에서 코딩을? 불편하지 않나?" 싶었는데, 써보면 생각이 달라집니다.

IDE 없이도 프로젝트 전체를 이해하고 작업하는 게 가능하고, 특히 **복잡한 리팩토링**이나 **여러 파일에 걸친 변경**에서 강점을 발휘해요.

## 실제로 어떤 상황에서 뭘 쓰나

솔직히 말하면 지금은 이렇게 나눠 씁니다:

**Cursor 쓸 때**
- 코드를 직접 보면서 수정할 때
- 한 파일에서 집중적으로 작업할 때
- 자동완성이 필요한 반복 코딩

**Claude Code 쓸 때**
- "이 기능 전체 다 만들어줘" 같은 큰 작업
- 프로젝트 구조를 처음부터 잡을 때
- 내가 직접 보지 않아도 되는 보일러플레이트 작업

## 가격은?

Cursor는 월 $20 (Pro 플랜). Claude Code는 Anthropic API 사용량에 따라 과금됩니다. 많이 쓰면 Claude Code가 더 비쌀 수 있는데, 무거운 작업에만 쓰면 오히려 저렴합니다.

둘 다 쓰는 제 기준으로는 한 달에 $30~40 정도 나오는 것 같아요. 생산성 대비 충분히 납득되는 금액이라고 생각합니다.

## 결론

뭐가 더 낫냐고 물으면, **용도에 따라 다르다**고 답합니다. 두 툴은 겹치는 부분도 있지만 근본적으로 다른 경험을 제공해요. 가능하면 둘 다 써보고 자기한테 맞는 걸 찾는 게 맞는 것 같습니다. 다음 글에서는 실제 프로젝트에서 두 툴을 어떻게 함께 쓰는지 구체적으로 적어볼게요.
